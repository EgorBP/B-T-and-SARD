## Технологический стек

**Frontend:** чистый JavaScript + Jinja2.

* Шаблоны Jinja2 позволяют отображать динамический контент (списки треков, карточки пользователей).
* Состояние пользователя хранится в сессии, навигация между страницами осуществляется через обычные ссылки и формы.

**Backend:** Python + FastAPI.

* Python выбран за простоту и удобство для веб-разработки.
* FastAPI отвечает за обработку запросов, работу с базой данных и хранение файлов, а также обеспечивает маршруты для API и страниц.
* База данных - SQLite. Используется ORM SQLAlchemy для работы с моделями User и Track.

**Инфраструктура:**

* Проект можно развернуть на Render, Heroku или PythonAnywhere.
* База данных хранится локально (SQLite) или в облаке (PostgreSQL).
* Загруженные аудиофайлы хранятся в папке media или облачном хранилище (например, AWS S3).
